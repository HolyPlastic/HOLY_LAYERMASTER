# Search & Hunt

**Files:** `js/main.js` (listener, HLMCache module) | `jsx/hostscript.jsx` → `searchLayersB64()`, `huntLayers()`, `hlm_buildLayerMetadataJSON()`, `hlm_selectExpressionLinks()` | `js/main_PICKCLICK.js` | `jsx/Modules/host_PICKCLICK.jsx`
**UI Section:** Search input (top) + Hunt tab (dimensions, pick-click buttons, SELECTA lineage)

## Search
Selects layers in the active comp whose names **or comments** contain the search string (case-insensitive). Fires on button click only — no live filtering.

Search term is Base64-encoded in JS before passing to ExtendScript via `evalScript` to avoid quote-escaping/injection issues. Custom `Base64.decode` used in JSX (native `atob` unavailable in ExtendScript).

Clears existing selection before applying result.

## Hunt Tab — PickClick (Parent / Children / Track Matte)
Three dimension inputs (Parent / Children / Track Matte) can now be filled by clicking a layer in the AE timeline:
- Each dimension row shows a text input (layer name, editable) + a pick-click trigger button (spiral+cursor icon).
- Clicking the trigger arms pick-click mode — the full-panel veil (`#hlmPickClickVeil`) appears.
- The JSX host polls every 250ms (`hlm_PC_poll`) for a change in `comp.selectedLayers`. When the user clicks a layer in AE, the layer name is dispatched via CSXSEvent `com.holy.layermaster.pickclick.resolve` and written into the dimension input.
- Clicking the veil (or waiting 10 s — frontend `SAFETY_TIMEOUT_MS` + backend `HLM_PICKCLICK_MAX_POLL_TICKS = 40 * 250ms`) cancels pick-click. See Dev Log entry 9.
- **Files:** `js/main_PICKCLICK.js` (frontend, namespace `Holy.LayerMaster.PickClick`) | `jsx/Modules/host_PICKCLICK.jsx` (backend, `hlm_PC_armPickClick`, `hlm_PC_cancelPickClick`, `hlm_PC_poll`)
- **Events:** `com.holy.layermaster.pickclick.resolve` / `.cancel` / `.trace`

## SELECTA — Select Expression Links
Button "EXPR LINKS" selects all layers referenced in expression strings on any currently selected layer's properties. Recursively walks the property tree, parses `layer("Name")` / `thisComp.layer("Name")` / `layer(N)` patterns, and adds matched layers to the selection (additive — does not deselect current).
- **JSX function:** `hlm_selectExpressionLinks()` in `hostscript.jsx`
- **JS listener:** `selectExprLinksBtn` click → `evalScript('hlm_selectExpressionLinks()')`

## Metadata Cache
`HLMCache` (JS-side module in `main.js`) builds a one-pass snapshot of all layers on every comp/project context change. Cache shape: `{ [layerId]: { id, index, name, comment, shy, solo, locked, enabled, parentId, label } }`.
- Built by `hlm_buildLayerMetadataJSON()` in `hostscript.jsx` (single `comp.numLayers` pass).
- Invalidated and rebuilt via `HLMCache.build(compId)` inside `_applyContext()`.
- Staleness guard: `HLMCache.isValid(liveNumLayers)` checks cached count vs live count.
- `searchLayersB64` extended to match name OR comment in one pass (no separate iteration).
