# Memory Banks

**Files:** `js/main.js` → `renderBankRow()`, `captureData()`, `selectData()`, `addBank()`, `removeBank()`
**UI Sections:** KF Memory, Layer Memory

Named banks for saving and recalling selections. Config (names, icons, colors) is global across projects; data is per-composition. Banks can be added/removed dynamically; order is drag-reorderable (see `docs/features/08-drag-drop.md`).

**UI structure per bank:** stacked [Select icon btn / Capture btn] + name input + clear btn.
- Select btn: always solid with bank color.
- Capture btn: accent-outlined when bank has data for the current comp. Tooltip shows saved item count.
- Right-click either Select or Capture → context dropdown: Clear (same as ×) | Colours (opens color picker).

For DNA tagging and storage layout see `docs/ARCHITECTURE.md`.

## 2.1 Keyframe Memory Banks

**JSX:** `captureKeyframes()`, `selectKeyframesFromFile()`, `hlm_readKeyFingerprint()`, `hlm_buildPropSequence()`, `hlm_scoreKey()`, `hlm_findSequenceOffset()`, `getPropPath()`, `resolvePropPath()`

**Capture** stores per captured property: `layerId`, `instanceId` (DNA), `propPath` (matchName chain), `keys` (1-based indices selected at capture), and `propSequence` — a full fingerprint of every key on that property: `{t, v, ii, oi}` (time, value, in/out interpolation type). Top-level: `compId`, `bankId`, `frameDuration`, `schemaVersion: 2`. One DNA tag per unique layer per capture; all properties on that layer share the tag.

**Recall — fuzzy-match confidence scoring.** The old index-locked recall was brittle: adding one keyframe before a banked block shifted every index and silently selected the wrong keys. The new system ignores indices and scores every current keyframe against each banked fingerprint, then picks the highest scorer per banked key.

Per-key score (0–100):
- **Time match** (within half a frame): +40
- **Value match** (scalar or numeric-array, ε=1e-4): +30 (+15 partial credit for non-scalar values like mask paths, which can't be compared)
- **Interpolation match** (both in + out type): +20
- **Neighbor-delta match** (time distance from previous banked key, preserved under uniform retimes): +10
- **Sequence-alignment bonus** (+15): a single dominant time-offset Δt is derived once per property by checking which Δ aligns the most banked keys onto current keys. Candidates landing at the expected `bankedTime + Δ` get bonus points — this disambiguates copy-pasted identical keyframe blocks that have been retimed.

**Thresholds:**
- Fuzzy mode (default, `kfStrictMatch: false`): accept ≥55.
- Strict mode (`kfStrictMatch: true`, via KF header padlock toggle): accept ≥95 — essentially exact matches only.
- **Ambiguity guard:** if top and second candidate are within 10 points *and* both clear threshold, refuse the match. Counted as `ambiguous` in the report.

**Report surfaced via toast** (`_showKfRecallToast` in `main.js`) above the banks grid: `{total, exact, shifted, skipped, ambiguous}`. Shifted = matched but not perfect (e.g. value tweak, time shift). Skipped = below threshold or ambiguous. Toast auto-dismisses after ~4s, colored ok/warn/err by outcome.

**Backward compatibility:** banks written before schema v2 have no `propSequence` — the recall path falls back to legacy index-only selection for those records automatically (detected per-keyframe-record, not per-bank).

**Performance:** capture cost is one extra loop of ~4 field reads per key on each captured property (sub-millisecond for typical mograph; ~200–500ms worst-case for 2000-key baked tracking data). Recall scores `numKeys × bankedKeys` per property, capped at 20×60 for the sequence-alignment pass. Button-press event, not a hot path.

> Silent-failure mode ("wrong keys selected, high confidence") requires a near-duplicate fingerprint elsewhere on the same property *and* sequence alignment agreeing — in practice the ambiguity guard catches these.

## 2.2 Layer Memory Banks

**JSX:** `captureLayers()`, `selectLayersFromFile()`

Captures selected layers by `layer.id` + DNA `instanceId`. Recall: O(1) by `layer.id` first, DNA tag scan fallback.

Backward compat: old `data.ids` flat array is promoted to `data.layers` format transparently on recall.
